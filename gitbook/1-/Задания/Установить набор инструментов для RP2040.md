# Папка для разработки и Pico-SDK

1. Создаем папку для разработки прошивки и переходим в нее:

``` bash
# Для MacOS и Linux переходим в директорию пользователя
cd ~/Repositories/pico

# Для Windows переходим на диск С: (либо любой другой), чтобы
# избежать кирилических символов в пути
cd C:/Repositories/pico
```

> [!ATTENTION] Внимание!
> 
> Для этого курса (и для любой другой разработки) важно работать в директории без кирилических символов и длинного пути, чтобы не возникали ошибки при сборке. Для Windows лучше всего использовать корень диска `С`, для Linux MacOS папку пользователя `~`. Все проекты принято размещать в папке `Repositories`, а всю работу по курсу надо разместить в папке `pico`.

2. Клонируем набор инструментов и библиотек pico-sdk для разработки прошивок под микроконтроллеры Raspberry:

``` bash
# Находимся в папке Repositories/pico
git clone -b master https://github.com/raspberrypi/pico-sdk.git
```

3. Переходим в папку `pico-sdk` и инициализируем сабмодули этого репозитория:

``` bash
# Находимся в папке Repositories/pico
cd pico-sdk
git submodule update --init
```

4. Проверить, что в результате вышеописанных действий на компьютере, в папке pico, появилась папка `pico-sdk`, содержащая весь SDK. В ней мы ничего не меняем, это набор инструментов от производителя pico для создания прошивок под их микроконтроллеры:

```
Repositories
|-- pico
	|-- pico-sdk
```

5. Создать системную переменную `PICO_SDK_PATH` со значением пути к папке `pico-sdk` (в примере это `C:\Repositories\pico\pico-sdk\`). Как это сделать указано здесь: [[Как создать системную переменную]].

> [!ATTENTION] Внимание!
> 
> Убедитесь, что вы не забыли этот пункт! Добавление переменной среды обязательно для сборки прошивки.
