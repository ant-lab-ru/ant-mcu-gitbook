# Результат

- [ ] В корне репозитория `mcu` находится `.gitignore` файл;
- [ ] В `.gitignore` добавлен шаблон на игнорирование всех директорий с именем `build/` системой контроля версий Git;
- [ ] В `.gitignore`  добавлен шаблон на игнорирование всех директорий с именем `.vscode/` системой контроля версий Git;
- [ ] Сделаны `commit` и `push` файла `.gitignore` в репозитории `mcu`

> [!INFO] Справка
> 
> [[Что такое .gitignore]]? 

# Инструкция

1. Создать в корне вашего репозитория `mcu` пустой текстовый файл с именем:

```
.gitignore
```

> [!ATTENTION] Обратите внимание!
>
>  Имя файла должно быть в точности `.gitignore`, т. к. это зарезервированное имя файла для системы git. В этом файле хранятся шаблоны имен файлов и папок, которые git должен игнорировать. Такие файлы не будут отображаться в `git status` и не будут попадать в `git add`.

2. Открыть файл `.gitignore` в VSCode и добавить в него строчки:

```
build/
.vscode/
```

> [!INFO] Примечание
> 
> В дальнейшем, в каждом проекте мы будем создавать папку `build` и собирать прошивки именно в ней. Во время сборки в этих папках будет множество временных файлов, которые **не должны** попадать репозиторий. Именно поэтому мы добавляем `build` в `.gitignore`.

> [!INFO] Примечание
> `.vscode/` - папка конфигураций VSCode, ее коммитить тоже **не нужно**.

4. Добавить файл `gitignore` в index (сделать `git add`), сделать `commit`, `push`:

``` bash
git add .gitignore
git commit -m "Added .gitignore file"
git push
```
