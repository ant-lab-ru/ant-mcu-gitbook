
`.gitignore` — это текстовый файл, содержащий список имен или шаблонов файлов и папок, которые система контроля версий Git должна игнорировать и не отслеживать. Он предотвращает попадание в репозиторий ненужных временных файлов, логов, секретных ключей или папок сборки, обеспечивая чистоту проекта.

`.gitignore` поддерживает шаблоны, поэтому, например, добавления строчки `build/` достаточно, чтобы исключить все папки `build` всех проектов нашего репозитория из системы версионирования Git. 

Для разных проектов и языков (C, C++, Python, JS и т. д.) есть набор типовых шаблонов для исключений, в которые добавляются свойственные языку и системе сборки кэши, папки сборки, временные файлы и т.д. Зачастую, используются стандартные шаблоны, которые может сгенерировать, например, сайт github. В нашем курсе используется фундаментальная система сборки через `cmake` и `make` без лишних инструментов, поэтому достаточно `.gitignore` состоящего всего из двух строчек.
