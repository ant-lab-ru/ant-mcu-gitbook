**Переменная среды** (англ. environment variable) — это текстовая переменная операционной системы, которая хранит информацию о конфигурации системы. Представляет собой пару **«имя — значение»**, где оба элемента — текстовые строки. Перемнные среды - общая концепция для всех основных операционных систем (Linux, Windows, MacOS).

Одна из основных задач, для которых используются переменные среды - это хранить на путь к исполняемым файлам, которые установлены в системе. Для чего и когда это нужно, опишем несколько примеров:
1. На компьютере установлена программа git. Она установлена без ошибок, работает, но, когда мы пытаемся вызвать ее через терминал (например, пишем git pull), то терминал пишет ошибку, что git - неизвестное имя. Почему так происходит? Потому что операционная система и терминала не знают о том, что вы установили программу, под названием git. Обычная установка - это просто копирование разных файлов на компьютер, для системы это не отличается от скачивания фотографии из интернета. **Для путей к исполняемым файлам используется переменная среды со стандартным именем PATH. Эта переменная уже существует и в нее надо **добавить** значение пути к папке с исполняемым файлом** git (например, `C:\Program Files\Git\cmd`). Разумеется, на разных системах этот путь может быть разным и его надо определить и проверить через проводник, определив где действительно находится искомый исполняемый файл.
2. Хочу скомпилировать прошивку, использующую набор инструментов pico-sdk. Запускаю сборку, но cmake выдает ошибку, что не найдена переменная среды PICO_SDK_PATH. Почему так происходит? Для сборки прошивки требуются различные файлы в папке pico-sdk, но она может быть расположена где угодно на компьютере либо отсутствовать, если мы ее еще не склонировали. Поэтому, при инициализации pico-sdk, скрипт `pico_sdk_import.cmake` требует, чтобы пользователь задал переменную среды с именем PICO_SDK_PATH, в значении которой будет путь к папке pico-sdk, содержащей необходимые библиотеки и инструменты. **Здесь мы уже создаем новую переменную среды с именем PICO_SDK_PATH** и значением, например (`C:\Repositories\pico\pico-sdk`).
